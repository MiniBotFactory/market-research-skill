# 竞品分析工作流

完整的竞品分析项目执行流程，约需 3-7 个工作日。

---

## 适用场景

- 评估是否进入某市场
- 制定差异化产品策略
- 定价策略制定
- 融资材料的竞争格局章节
- 年度战略规划

---

## 执行流程

### 阶段 0：需求确认（0.5天）

**角色**：需求分析师

必须明确：
- [ ] 调研的竞品范围（哪些公司？按什么标准筛选？）
- [ ] 核心对比维度（产品？定价？市场份额？）
- [ ] 调研目的（进入决策/差异化策略/定价参考）
- [ ] 交付物格式（矩阵表格/详细报告/PPT）

**输出**：竞品分析需求确认单

---

### 阶段 1：竞品清单确定（0.5天）

**角色**：需求分析师 + 项目经理

建立竞品分析边界：

```
第一梯队（直接竞品）：相同目标客群 + 相似核心功能
第二梯队（间接竞品）：同一问题的替代解决方案
第三梯队（潜在竞品）：有能力进入的大公司
```

筛选原则：
- 直接竞品：聚焦 3-8 个（可深度分析）
- 间接竞品：最多 5 个（仅概述）
- 建议向客户确认最终竞品清单

---

### 阶段 2：数据收集（1-3天，并行）

**角色**：数据研究员（内部）+ 市场情报员（外部）— 同时启动

**数据研究员任务清单**：
- [ ] 检索内部是否有相关竞品的历史分析
- [ ] 整理内部同事对竞品的已知信息
- [ ] 汇总内部客户提及竞品的反馈记录

**市场情报员任务清单**（对每个直接竞品）：
- [ ] 基本信息：成立时间、融资、规模（企查查/Crunchbase）
- [ ] 产品信息：核心功能列表、近期发布（官网/App Store）
- [ ] 定价信息：定价页面截图/记录（官网定价页）
- [ ] 客户信息：目标客群、标杆案例（官网案例）
- [ ] 口碑信息：用户评价、常见抱怨（评论平台）
- [ ] 近期动态：最近3个月重要事件（新闻/公告）

**数据来源优先级**：参见 `references/methods/data-sources.md`

---

### 阶段 3：分析（0.5-1天）

**角色**：首席分析顾问

使用 `references/methods/competitor-analysis.md` 中的框架：

**必须完成**：
1. 竞品对比矩阵（所有竞品横向对比）
2. 竞品定位图（2D坐标，识别空白）
3. TOP 3 差异化机会（市场空白 or 竞品弱点）

**选择性完成**（根据调研目的）：
- 如果是「进入决策」→ 补充波特五力分析
- 如果是「差异化策略」→ 补充 SWOT 分析（针对自身）

---

### 阶段 4：报告起草（0.5天）

**角色**：项目经理

使用 `assets/report-templates/competitor-analysis-report.md`

标准竞品分析报告结构：
```
1. 执行摘要（关键发现 + 建议，1页）
2. 竞争格局概览（竞品清单 + 市场结构）
3. 竞品深度分析（对比矩阵 + 各竞品详情）
4. 差异化机会（空白市场 + 我方优劣势）
5. 战略建议（具体可执行，按优先级）
6. 附录（数据来源 + 置信度说明）
```

---

### 阶段 5：质量审核（0.5天）

**角色**：质量审核员

竞品分析特别关注点：
- [ ] 竞品清单是否经过客户确认？
- [ ] 定价数据是否有截图/时间戳证明时效性？
- [ ] 是否忽略了重要的间接竞品？
- [ ] 差异化建议是否基于数据而非主观判断？

---

## 快速版（1-2天）

当时间紧迫时的精简版：

```
Day 1：市场情报员快速扫描 3-5 个核心竞品
Day 2：首席分析顾问完成竞品矩阵 + 2-3条建议
交付：单页竞品对比表 + 执行摘要
```

适用：需要快速决策的场合，明确告知客户这是「快速扫描」而非「深度研究」。

---

## 常见问题处理

**问题：竞品不公开定价怎么办？**
→ 记录为「联系销售报价」，补充通过试用/演示获得的价格线索，或引用行业媒体报道的定价区间

**问题：竞品是未上市私有公司，无财务数据**
→ 用员工规模（LinkedIn）、融资轮次（Crunchbase）、产品定价反推估算规模

**问题：竞品太多，无法全部深度分析**
→ 确认客户最关心的 3-5 个竞品做深度分析，其余做概述对比
