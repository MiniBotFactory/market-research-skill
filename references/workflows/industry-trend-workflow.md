# 行业趋势分析工作流

行业趋势报告的执行流程，约需 3-7 个工作日。

---

## 适用场景

- 年度行业报告
- 投资赛道研究
- 战略规划的环境扫描（PESTEL）
- 行业白皮书

---

## 执行流程

### 阶段 0：需求确认（0.5天）

**角色**：需求分析师

关键明确事项：
- [ ] 行业边界（如何定义这个行业？）
- [ ] 时间维度（过去3年趋势 vs 未来3年预测）
- [ ] 重点趋势类型（技术/市场/政策/消费）
- [ ] 受众（投资人/行业从业者/政策制定者）

---

### 阶段 1：趋势框架搭建（0.5天）

**角色**：需求分析师 + 首席分析顾问

使用 PESTEL 框架确保全面覆盖：

```
P - Political（政策/监管）：政府政策、监管变化、产业支持
E - Economic（经济）：宏观经济、行业增速、资本流向
S - Social（社会）：人口结构、消费习惯、文化变迁
T - Technological（技术）：新技术应用、技术成熟度
E - Environmental（环境）：可持续发展、ESG趋势
L - Legal（法律）：合规要求、知识产权、数据安全
```

根据调研目的，筛选最相关的 3-5 个趋势维度重点研究。

---

### 阶段 2：数据收集（2-4天，并行）

**角色**：数据研究员 + 市场情报员 — 同时启动

**市场情报员重点来源**：

**政策趋势**：
- 政府工作报告、五年规划
- 工信部、发改委行业政策
- 行业协会年会纪要

**技术趋势**：
- Gartner Hype Cycle（技术成熟度曲线）
- 学术论文引用量（Google Scholar）
- 行业顶级会议（看什么在被讨论）
- 头部公司研究院报告

**市场趋势**：
- 行业融资数据（方向反映趋势）
- 龙头企业战略动作（并购/招聘/投资）
- 消费行为数据（电商、搜索趋势）

**数据收集优先级**：见 `references/methods/data-sources.md`

---

### 阶段 3：趋势分析（1天）

**角色**：首席分析顾问

**趋势评估框架**：

对每个识别的趋势，评估三个维度：

| 维度 | 说明 | 评估方法 |
|------|------|----------|
| **确定性** | 这个趋势发生的确定程度 | 已有数据支撑 vs 推测 |
| **影响力** | 对行业/企业的影响程度 | 颠覆性/重大/边际 |
| **时间窗口** | 何时会产生实质影响 | 1年内/1-3年/3年以上 |

**趋势优先级矩阵**：
```
高确定性 + 高影响力 + 近期 → 必须应对（现在行动）
高确定性 + 高影响力 + 远期 → 战略部署（提前规划）
低确定性 + 高影响力 → 持续关注（监测信号）
低影响力 → 低优先级（定期回顾即可）
```

**趋势相互关系分析**：
- 哪些趋势相互加强？（形成「趋势浪潮」）
- 哪些趋势相互冲突？（带来不确定性）

---

### 阶段 4：报告起草（0.5天）

**角色**：项目经理

行业趋势报告结构：
```
1. 执行摘要（3-5个最重要的趋势，1页）
2. 行业概况（规模、参与者、近年变化）
3. 核心趋势分析（每个趋势：现状+数据+未来走向+影响）
4. 机会与威胁（趋势对受众的含义）
5. 未来3-5年情景展望（乐观/基准/悲观）
6. 附录（数据来源、方法说明）
```

---

### 阶段 5：质量审核（0.5天）

**角色**：质量审核员

**行业趋势报告特别关注**：
- [ ] 趋势是否有数据支撑，还是仅凭直觉？
- [ ] 时间预测是否有明确说明「这是预测，不是确定」？
- [ ] 是否识别了反向的风险信号（不只讲正面趋势）？
- [ ] 结论是否对受众有实际的决策指导价值？
