# 用户研究工作流

用户访谈和问卷调研的完整执行流程，约需 2-4 周（含招募时间）。

---

## 适用场景

- 产品立项前的需求验证
- 产品迭代的用户痛点挖掘
- 竞品切换原因分析
- 用户画像构建

---

## 执行流程

### 阶段 0：研究设计（0.5-1天）

**角色**：需求分析师

明确研究设计的五个要素：

**1. 核心研究问题**（最重要）
```
不超过 3 个核心问题，例如：
- 用户为什么选择/不选择我们的产品？
- 用户在[场景]中最大的痛点是什么？
- 用户对[新功能]的接受度如何？
```

**2. 研究方法选择**
- 需要发现未知需求 → 深度访谈（定性）
- 需要量化验证 → 在线问卷（定量）
- 两者兼顾 → 先访谈再问卷

**3. 目标受访者画像**
```
主要：[职位/行业/使用场景]
次要：[对照组，如当前不是用户]
排除：[不需要的用户类型]
```

**4. 样本量**
- 深度访谈：6-15人（每个细分群体至少5人）
- 在线问卷：最少 100 人，推荐 200-500 人

**5. 时间计划**
```
Week 1：研究设计 + 招募
Week 2-3：数据收集（访谈/问卷）
Week 4：分析 + 报告
```

---

### 阶段 1：招募受访者（3-7天）

**角色**：项目经理 + 市场情报员

招募渠道（按可及性排序）：
1. 内部客户/潜在用户数据库
2. 业务团队推荐（注意样本偏差）
3. 社交媒体招募（LinkedIn/行业微信群）
4. 第三方招募平台（问卷星、见数）
5. 外部调研公司（成本高但效率高）

**筛选问卷（必须）**：
- 确认目标用户身份
- 排除竞品公司员工
- 确认可用时间

---

### 阶段 2：数据收集（1-2周）

**角色**：市场情报员（执行访谈/问卷）

参考详细方法：`references/methods/user-research.md`

**访谈执行清单**：
- [ ] 提前发送访谈确认邮件（含时间/形式/目的）
- [ ] 获得录音/记录同意
- [ ] 使用标准访谈指南（允许灵活发挥，但覆盖所有核心问题）
- [ ] 每次访谈后立即整理笔记（记忆会快速消退）
- [ ] 每3-4次访谈后回顾，看是否需要调整问题

**问卷发布清单**：
- [ ] 小范围测试（5-10人预测试）
- [ ] 检查逻辑跳转是否正确
- [ ] 设置填写时限（避免随意作答）
- [ ] 设置甄别题放在最前面

---

### 阶段 3：数据分析（2-4天）

**角色**：首席分析顾问

**定性分析（访谈数据）**：

步骤 1：原始数据整理
- 将录音转录为文字（可用讯飞听见等工具）
- 整理每次访谈的关键语录和发现

步骤 2：主题聚类
- 使用亲和图法，将所有洞察按主题分组
- 识别重复出现的模式（出现≥3次即为主要主题）

步骤 3：优先级排序
- 按「重要性 × 频率」排序痛点
- 识别「高频高烈度」的核心痛点

**定量分析（问卷数据）**：
- 基础统计（均值/中位数/分布）
- 按细分群体的交叉分析
- NPS 计算和解读
- 开放题主题提取

---

### 阶段 4：用户画像构建（0.5天）

**角色**：首席分析顾问

基于数据将用户分为 2-3 类主要画像，每类用模板（`references/methods/user-research.md`）描述。

**画像验证**：
- 画像应该互相排斥（不同类之间有明显差异）
- 画像应该覆盖全面（所有受访者能归入某类）
- 画像应该可操作（不同画像有不同的产品/市场策略含义）

---

### 阶段 5：报告与审核（1天）

**角色**：项目经理 → 质量审核员

**用户研究报告必须包含**：
- [ ] 研究方法说明（样本量/招募方式/访谈时间）
- [ ] 用户画像（2-3个）
- [ ] TOP 5 核心痛点（按优先级）
- [ ] 代表性用户引语（支撑关键发现）
- [ ] 对产品/业务的具体建议

**质量审核特别关注**：
- [ ] 样本量是否足够？是否有明显的选择偏差？
- [ ] 是否区分了「用户说的」和「分析师的解读」？
- [ ] 用户引语是否真实，是否脱离了上下文？
- [ ] 结论是否基于多个受访者的共同反映，而非个别案例？
