# 用户研究方法论

本文档提供用户调研的核心方法和操作指南，供执行用户研究时使用。

---

## 方法选择指南

| 方法 | 最适合 | 样本量 | 时间 | 成本 |
|------|--------|--------|------|------|
| 深度访谈 | 理解用户动机、发现未知需求 | 8-15人 | 2-4周 | 高 |
| 焦点小组 | 验证概念、获取即时反应 | 6-8人/组 | 1-2周 | 中 |
| 在线问卷 | 量化验证、用户画像数据 | 100-500+ | 1-2周 | 低 |
| 用户日记/观察 | 真实使用行为、隐性需求 | 5-10人 | 2-6周 | 高 |
| 竞品用户访谈 | 了解竞品体验、切换动机 | 5-10人 | 1-2周 | 中 |

**快速调研（1周内）**：在线问卷 + 3-5个目标用户电话访谈
**深度调研（1个月）**：深度访谈 + 问卷量化验证

---

## 方法 1：深度用户访谈

### 访谈设计原则

**开放性**：避免是/否问题，用开放式问题引导用户说出真实想法
**中立性**：不透露自己的观点，不引导用户说「正确答案」
**追问**：对有价值的回答追问「为什么」「能举个例子吗」
**场景化**：让用户描述具体场景，不要讨论抽象概念

### 访谈结构（60分钟）

```
00-05 分钟：暖场
  - 感谢参与，说明用途（匿名/研究目的）
  - 请用户简单介绍自己（工作/角色）

05-15 分钟：背景了解
  - 工作场景和日常流程
  - 目前使用的工具和方式

15-45 分钟：核心问题探索
  - 围绕调研主题的核心问题（5-8个）
  - 追问具体案例和场景

45-55 分钟：痛点深挖
  - 最大的挑战/挫折感
  - 理想解决方案是什么样的

55-60 分钟：结尾
  - 还有什么想补充的？
  - 是否愿意推荐其他用户参与？
```

### 核心问题设计模板

```
背景问题：
- "请描述一下您日常处理[主题]的典型工作流程？"
- "您目前使用什么工具/方法来解决[问题]？"

深挖问题：
- "上次您遇到[挑战]是什么情况，您是如何处理的？"
- "这个过程中，哪个环节是最让您头疼的？为什么？"
- "如果这个问题解决不了，会对您的工作造成什么影响？"

需求发现：
- "如果有一个完美的解决方案，它应该是什么样的？"
- "您有没有临时想出的办法，虽然不完美但凑合用？"
- "您见过其他人用更好的方法解决这个问题吗？"
```

## 方法 2：在线问卷

### 问卷结构设计

```
第1部分：筛选题（2-3题）
  目的：确保受访者是目标用户
  示例：「您是否使用过[产品类型]？」是→继续 否→结束

第2部分：背景信息（3-5题）
  目的：用户画像数据
  示例：公司规模、职位、行业、使用频率

第3部分：核心问题（5-10题）
  目的：量化需求和痛点
  类型：量表题（1-5/1-10）+ 选择题

第4部分：开放题（1-2题）
  目的：获取定性洞察
  示例：「您觉得现有工具最大的不足是什么？」

第5部分：意愿题（1-2题）
  目的：评估产品需求
  示例：「如果有一款满足X需求的产品，您愿意为此付费吗？」
```

### 量表题最佳实践

**NPS（净推荐值）**：
```
「您有多大可能向同事推荐[产品/服务]？」
0（完全不会推荐）—— 10（极力推荐）
```

**痛点严重程度评估**：
```
「以下问题对您日常工作的影响程度？」
1=几乎没影响  3=有一定影响  5=严重影响，必须解决
```

**付费意愿评估（Van Westendorp 定价法）**：
```
1. 这个价格便宜到让您觉得质量有问题：____元
2. 这个价格便宜但您还能接受：____元
3. 这个价格有点贵但您还能接受：____元
4. 这个价格太贵，您绝对不会买：____元
```

## 数据分析方法

### 定性数据分析（访谈）

**亲和图法（Affinity Diagram）**：
1. 将每条访谈洞察写在便利贴上（或表格行）
2. 按主题分组聚类
3. 为每个组命名（即用户需求类别）
4. 按频次和严重程度排序

**需求优先级矩阵**：
```
重要性（用户认为多重要）× 满意度（现有方案满足程度）

高重要-低满意 = 核心机会（优先解决）
高重要-高满意 = 维持优势（不要破坏）
低重要-低满意 = 次要问题（有余力再做）
低重要-高满意 = 资源浪费（考虑精简）
```

### 定量数据分析（问卷）

**交叉分析**：
- 不同用户群体的需求差异（按职位/公司规模/行业）
- 识别「重度用户」和「普通用户」的行为差异

**净推荐值计算**：
```
NPS = 推荐者（9-10分）% - 贬损者（0-6分）%
  > 50：极好  30-50：好  0-30：一般  < 0：有问题
```

```markdown

### 基本信息
- 职位：[...]
- 公司规模：[...]
- 行业：[...]
- 年龄段：[...]

### 目标与动机
- 工作目标：[他们工作中想要实现什么]
- 个人动机：[什么对他们重要]

### 核心痛点（按严重程度排序）
1. [痛点 1]：[具体描述]
2. [痛点 2]：[...]

### 现有解决方案
- 目前使用的工具/方法：[...]
- 不满意的地方：[...]

### 典型使用场景
「[一段描述这类用户在典型场景下行为的小故事]」

### 影响决策的关键因素
- [购买/采用的主要考量因素]

### 代表性引语
"[来自访谈的真实原话，最能代表这类用户的声音]"
```
