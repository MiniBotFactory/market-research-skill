# 定价研究方法论

本文档提供定价策略制定所需的分析框架和研究方法，包括价格敏感度测量、竞争性定价分析、定价模式设计。

## 方法 1：Van Westendorp 价格敏感度测量（PSM）

**最常用**：适合获取产品的可接受价格区间，成本低、操作简单。

### 四个核心问题

在问卷中对目标用户提问：

```
Q1. 您觉得什么价格是便宜到让您觉得产品质量可能有问题？
    → 价格过低点（Too Cheap）

Q2. 您觉得什么价格是便宜且物有所值的？
    → 可接受低价点（Cheap/Bargain）

Q3. 您觉得什么价格有点贵，但如果产品很好您还是会考虑购买？
    → 可接受高价点（Expensive）

Q4. 您觉得什么价格是贵到无法接受、绝对不会购买的？
    → 价格过高点（Too Expensive）
```

### 分析方法

将四条线绘制在同一坐标系（X轴=价格，Y轴=累积%）：

```
% 受访者

100│             ╲ Q4（太贵）
   │              ╲
   │  Q1（太便宜）  ╲___
   │  ___/            ╲
   │ /    Q2（便宜）    ╲
   │      Q3（可接受贵） ╲
 0 │────────────────────────→ 价格
      ↑    ↑         ↑    ↑
     最低  可接受    可接受 最高
     可接受 低价点   高价点 可接受
     价格                  价格
```

**关键交叉点**：
- **可接受区间**：Q2 和 Q3 曲线之间的区域（最多人认为合理的价格范围）
- **最优价格点（OPP）**：Q1 和 Q4 曲线的交叉点（最多人接受，最少人觉得太贵/太便宜）
- **可接受价格点（IDP）**：Q2 和 Q3 曲线的交叉点（心理锚定价格）

### PSM 结果模板

```markdown
## Van Westendorp 定价分析结果

**产品**：[产品名称]
**样本量**：[N] 人（[目标客群描述]）

### 关键价格节点

| 节点 | 价格 | 含义 |
|------|------|------|
| 过低点（质量担忧） | ¥___ | 低于此价格，用户认为质量有问题 |
| 可接受低价点 | ¥___ | 物有所值的低端 |
| 最优价格点（OPP） | ¥___ | 综合接受度最高 |
| 可接受高价点 | ¥___ | 贵但可接受的上限 |
| 过高点（拒绝购买） | ¥___ | 超过此价格大多数人拒绝 |

### 可接受价格区间
¥___ ~ ¥___（约 [N]% 受访者的可接受范围）

### 定价建议
基于分析，推荐价格区间：¥___ ~ ¥___
定价逻辑：[...]
```

## 方法 2：Gabor-Granger 需求曲线

**适合**：测量不同价格点的购买意愿，绘制价格-需求曲线，估算最大化收益的价格点。

### 问卷设计

随机向不同用户展示不同价格（或向同一用户按降序展示）：

```
价格梯度示例：
¥99 → ¥149 → ¥199 → ¥299 → ¥399

每个价格点询问：
「如果该产品的价格是 ¥X，您购买的可能性是？」
□ 肯定会买（100%）
□ 很可能会买（75%）
□ 可能会买（50%）
□ 不太可能买（25%）
□ 肯定不买（0%）
```

### 分析

```
收益指数 = 购买意愿% × 价格

最大化收益指数的价格 = 最优定价点

示例：
| 价格  | 购买意愿% | 收益指数 |
|-------|-----------|---------|
| ¥99   | 72%       | 71.3    |
| ¥149  | 58%       | 86.4  ← 最高
| ¥199  | 41%       | 81.6    |
| ¥299  | 22%       | 65.8    |
| ¥399  | 11%       | 43.9    |
```

## 方法 3：基于价值的定价（Value-Based Pricing）

**适合**：差异化明显的产品，用来论证溢价空间。

### 四步法

**步骤 1：识别关键价值驱动因素**

通过用户访谈，找出产品带来的核心价值（量化优先）：
- 节省时间：每月节省 X 小时 × 用户时薪 = ¥Y
- 减少损失：避免的错误损失 = ¥Z
- 增加收入：带来的额外收入 = ¥W

**步骤 2：量化价值**

```
经济价值 = 参照方案成本 + 差异化价值

示例（项目管理SaaS）：
参照方案（Excel+邮件）的隐性成本：
  = 每周额外协调时间 3h × 5人 × ¥200/h × 52周
  = ¥156,000/年/团队

差异化价值（减少出错、更快交付）：
  估算 = 项目延期率降低 30% × 平均延期损失 ¥50,000
  = ¥15,000/年/团队

经济价值 = ¥171,000/年/团队
```

**步骤 3：估算用户愿意分享的价值比例**

通常产品可捕获价值的 10-20%：
```
建议定价 = ¥171,000 × 15% ≈ ¥25,650/年/团队
          ≈ ¥2,140/月/团队（约 ¥428/人/月）
```

**步骤 4：与市场价格对比**

```
| 定价方式 | 结果 |
|----------|------|
| 竞品均价 | ¥200/人/月 |
| 基于价值测算 | ¥428/人/月 |
| 用户访谈接受度 | ¥150-350/人/月 |
| 建议价格 | ¥249/人/月（竞品溢价25%，低于经济价值） |
```

## 方法 4：竞争性定价矩阵

**适合**：快速对标市场、制定差异化定价策略。

### 定价对比模板

```markdown
## 竞品定价矩阵：[产品类别]

| 竞品 | 定价模式 | 价格区间 | 免费层 | 企业版 | 目标客群 |
|------|----------|----------|--------|--------|----------|
| 竞品A | 按席位/月 | ¥X-Y/人/月 | 有/无 | 定制 | 中小企业 |
| 竞品B | 按使用量 | ¥X起 | 有 | ¥X/月 | 中大型 |
| 竞品C | 一次性买断 | ¥X | 无 | N/A | 个人/小团队 |
| 我方 | [待定] | [待定] | [待定] | [待定] | [待定] |

## 定价策略选项

**选项A：渗透定价**（低于市场均价20-30%）
- 目的：快速获取市场份额
- 风险：利润压力，品质感知低

**选项B：市场均价**（对标主要竞品）
- 目的：减少价格决策摩擦
- 风险：需要其他差异化维度

**选项C：溢价定价**（高于市场均价15-30%）
- 目的：树立高端品牌，提升利润
- 条件：有明确差异化价值支撑

## 推荐方向
[基于竞品分析和价值评估的具体建议]
```

## 定价模式选择指南

| 定价模式 | 适用场景 | 优点 | 风险 |
|----------|----------|------|------|
| **按席位/用户** | SaaS、团队工具 | 可预期收入，简单 | 客户会减少席位 |
| **按使用量** | API、数据服务 | 与价值挂钩 | 收入不稳定 |
| **分层套餐** | 覆盖多客群 | 扩大覆盖面 | 套餐设计复杂 |
| **一次性买断** | 单机软件、工具 | 心理门槛低 | 无持续收入 |
| **Freemium** | 高流量产品 | 低获客成本 | 转化率通常低 |
| **按结果/成效** | 咨询、效果广告 | 与客户利益一致 | 测量难度高 |

## 常见定价陷阱

| 陷阱 | 描述 | 规避方法 |
|------|------|----------|
| **成本加成** | 只看成本，不看价值和竞争 | 先做市场和用户调研 |
| **竞品对标** | 直接复制竞品定价，放弃差异化 | 结合价值定价测算 |
| **过度折扣** | 用折扣换量，破坏价格体系 | 设计阶梯定价替代折扣 |
| **一刀切** | 不同客群用同一价格 | 做客户细分，分层定价 |
| **忽略时机** | 定价后长期不调整 | 定期（每年）复查定价策略 |
