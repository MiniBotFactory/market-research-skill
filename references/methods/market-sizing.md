# 市场规模估算方法论

本文档提供 TAM/SAM/SOM 的计算方法和数据来源指南。

---

## 核心概念：TAM / SAM / SOM

```
TAM（Total Addressable Market）全球可寻址市场
  └── SAM（Serviceable Addressable Market）可服务的目标市场
        └── SOM（Serviceable Obtainable Market）可实际获得的市场份额
```

| 指标 | 定义 | 用途 |
|------|------|------|
| TAM | 如果100%市场份额，理论上的最大营收 | 展示赛道天花板，用于融资/战略 |
| SAM | 基于产品能力和地域限制，实际能服务的市场 | 资源分配，真实机会评估 |
| SOM | 基于竞争格局，未来3-5年可实际获取的份额 | 财务预测，投资回报估算 |

---

## 方法 1：自上而下（Top-Down）

**适用**：有行业总规模数据，需要分解到细分市场。

**步骤**：
```
全球/全国市场规模
    × 目标地域占比（%）
    × 目标行业渗透率（%）
    × 目标客群占比（%）
    × 平均客单价
    = 目标市场规模（SAM）
```

**示例**：
```
中国企业软件市场总规模：3,000亿元（来源：IDC 2023）
    × 中小企业占比：35%
    × HR软件渗透率：28%
    × 我司目标行业（制造业）占比：20%
    = SAM ≈ 588亿元
```

**常见数据来源**：
- 行业研究报告（IDC、Gartner、艾瑞等）
- 政府统计年鉴（国家统计局）
- 行业协会发布数据

**优点**：数据可引用，有外部背书  
**缺点**：容易高估，分子分母定义不精确时误差大

---

## 方法 2：自下而上（Bottom-Up）

**适用**：有用户/单价数据，从基础单元向上推算。

**步骤**：
```
目标客户总数
    × 愿意付费比例（%）
    × 平均年客单价（ARR）
    = 市场规模
```

**示例**：
```
中国制造业中小企业数量：约50万家
    × 有HR管理需求（50人以上）：20% → 10万家
    × 年付费意愿评估：30% → 3万家
    × 平均年费用：2万元/年
    = 市场规模 ≈ 6亿元
```

**数据来源**：
- 工商注册数据（企查查、国家市场监管总局）
- 行业调研/一手访谈
- 竞品已有客户数 × 市场份额估算反推

**优点**：基于实际单元，更接近真实  
**缺点**：需要准确的基础数据，获取成本高

---

## 方法 3：类比估算

**适用**：缺乏直接数据，但有可对比的参照市场。

**类比类型**：

**地域类比**：参考成熟市场推算新市场
```
美国某细分市场规模（已知）× 中美相关指标比例 = 中国市场估算
注意：选择合适的基准（GDP、人口、互联网渗透率、收入水平）
```

**时间类比**：参考发展曲线预测未来
```
参考行业S曲线，找到目标市场当前所处阶段，预测3-5年增长
```

**渗透率类比**：参考相似行业的渗透率
```
SaaS在制造业的渗透率 ≈ 已有SaaS在零售业的历史渗透率曲线
```

## 交叉验证（必须步骤）

使用两种方法分别估算后，进行交叉验证：

```
自上而下估算：X 亿元
自下而上估算：Y 亿元
差异：|(X-Y)/X| × 100%

< 20%：估算较可靠，取中间值
20-50%：检查假设，找出差异原因
> 50%：重新审视数据来源和假设，不宜直接引用
```

## 常见错误和规避方法

| 错误 | 规避方法 |
|------|----------|
| TAM/SAM 混用，声称 TAM 是自己的市场 | 明确区分三个层级，诚实说明限制条件 |
| 使用过时数据（3年以上）而不注明 | 所有数据标注年份，超过2年加说明 |
| 只用一种方法，没有交叉验证 | 至少使用两种方法 |
| 复制粘贴行业报告数字，不理解定义 | 理解报告的市场定义，确认是否与自己的一致 |
| 增速假设不合理（拍脑袋说 30% 增速） | 引用行业CAGR数据，或与GDP增速/行业相关指标挂钩 |
