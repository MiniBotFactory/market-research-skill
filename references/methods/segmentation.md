# 客户细分方法论

客户细分是将目标市场按共同特征分组，使营销、产品和销售策略更精准。**好的细分应该是「可识别、可衡量、可触达、有规模、有差异化需求」**。

---

## 细分维度选择

### B2C 市场细分维度

| 维度类型 | 常用变量 | 数据获取方式 |
|----------|----------|-------------|
| **人口统计** | 年龄、性别、收入、教育、家庭结构 | 问卷、用户注册信息 |
| **地理** | 地区、城市级别、气候 | 注册信息、IP数据 |
| **行为** | 购买频率、使用程度、渠道偏好 | 行为数据、交易记录 |
| **心理/价值观** | 生活方式、个性、价值取向 | 深度访谈、心理测量问卷 |

**优先级**：行为 > 心理 > 人口统计 > 地理

（行为和心理细分预测购买行为的准确性最高）

---

### B2B 市场细分维度

| 维度类型 | 常用变量 | 数据获取方式 |
|----------|----------|-------------|
| **公司特征** | 行业、规模（员工数/营收）、地域 | 工商信息、CRM |
| **运营特征** | 技术使用、采购模式 | 问卷、销售记录 |
| **采购特征** | 决策方式、优先级（价格/质量/服务） | 销售访谈 |
| **情境特征** | 使用场景、紧迫程度 | 访谈、客服记录 |
| **个人特征** | 决策者风险偏好、忠诚度 | 销售团队洞察 |

**B2B 最有价值的细分维度**：公司规模 × 行业 × 采购成熟度

## 方法 1：RFM 细分（行为细分，最常用）

**适合**：有交易数据的场景（电商、SaaS、零售）

**RFM 三个维度**：

| 维度 | 全称 | 含义 | 评分方式 |
|------|------|------|---------|
| **R** | Recency | 最近一次购买距今多久 | 越近 = 分越高（1-5）|
| **F** | Frequency | 购买/使用频率 | 越高 = 分越高（1-5）|
| **M** | Monetary | 消费金额 | 越高 = 分越高（1-5）|

### RFM 细分操作步骤

**Step 1：数据准备**
```
客户ID | 最近购买日期 | 购买次数（1年内） | 总消费金额
C001  | 2024-01-15  | 12              | ¥8,500
C002  | 2023-08-03  | 2               | ¥1,200
...
```

**Step 2：分位数打分（1-5）**
```python
# 伪代码示例
R_score = quantile_rank(days_since_last_purchase, reverse=True)  # 越近越高
F_score = quantile_rank(purchase_frequency)
M_score = quantile_rank(total_spending)
RFM_combined = R_score * 100 + F_score * 10 + M_score
```

**Step 3：定义客户群**

| 群体 | RFM 特征 | 策略 |
|------|----------|------|
| **冠军客户** | R高 F高 M高 (5,5,5) | 奖励、早期体验 |
| **忠实客户** | R中 F高 M高 | 升级会员、专属福利 |
| **潜力客户** | R高 F低 M中 | 提升频次、会员计划 |
| **即将流失** | R低 F高 M高 | 挽留活动、问卷 |
| **需挽回** | R低 F低 M低 | 激活活动 / 放弃 |
| **新用户** | R高 F低 M低 | 新手引导、首购优惠 |

## 方法 2：价值-需求二维细分（B2B 常用）

**适合**：B2B 企业制定差异化客户策略。

### 二维矩阵

```
高价值
  │  战略客户          关键增长客户
  │  (高价值-低需求波动) (高价值-高需求波动)
  │  → 深度绑定        → 满足差异化需求
  │
  ├─────────────────────────────────── 需求复杂度
  │
  │  效率客户          高潜力客户
  │  (低价值-低需求)    (低价值-高需求)
  │  → 自助服务        → 评估投入价值
低价值
        低复杂度              高复杂度
```

**四类客户策略**：

| 象限 | 策略重点 | 服务模式 | 定价 |
|------|----------|----------|------|
| 战略客户 | 深度合作，联合创新 | 专属客户成功 | 定制价格 |
| 关键增长 | 满足差异化需求 | 高触点服务 | 溢价 |
| 效率客户 | 降低服务成本 | 自助+标准化 | 标准价 |
| 高潜力 | 验证价值，决定投入 | 测试服务方案 | 协商 |

## 方法 3：行为聚类细分（基于数据的科学细分）

**适合**：有行为数据时（用户行为日志、使用数据）

### 聚类分析流程

**Step 1：选择聚类变量**

选择 3-7 个能区分用户群的行为指标：
```
示例（SaaS产品）：
- 月活跃天数
- 核心功能使用深度（1-5）
- 团队协作功能使用比例
- 数据导出频率
- 支持工单数量
```

**Step 2：执行聚类**

常用方法：K-means（需指定 K 值）、层次聚类（可视化树状图）

确定 K 值：肘部法则（Elbow Method）

**Step 3：命名和解读**

给每个群体起描述性名称，基于特征而非编号：

```
示例细分结果：
群体 A（28%）：深度用户 — 高频率、高深度、低工单 → "自给自足的专家"
群体 B（35%）：效率型 — 中频率、仅核心功能、高工单 → "被动使用者"
群体 C（20%）：团队型 — 低个人频率、高协作功能 → "工具推广者"
群体 D（17%）：边缘用户 — 低频率、浅使用 → "流失风险"
```

## 细分有效性验证标准

好的细分必须满足 5 个标准：

| 标准 | 检验问题 |
|------|----------|
| **可识别（Identifiable）** | 能否清晰描述每个细分的特征？ |
| **可衡量（Measurable）** | 能否估算每个细分的规模和价值？ |
| **可触达（Accessible）** | 能否通过营销渠道触达这些用户？ |
| **有规模（Substantial）** | 每个细分是否大到值得单独制定策略？ |
| **有差异（Differentiable）** | 不同细分对同一刺激的反应是否不同？ |

## 细分陷阱

| 陷阱 | 描述 | 规避方法 |
|------|------|----------|
| **细分过多** | 10个以上细分无法执行 | 控制在 3-6 个主要细分 |
| **只看人口统计** | 年龄/性别预测力弱 | 加入行为和需求维度 |
| **静态细分** | 用户会在细分间移动 | 定期（半年）重新细分 |
| **忽略内部资源** | 细分超出公司服务能力 | 先聚焦 1-2 个核心细分 |
| **沉没成本偏见** | 坚持低价值细分因为历史投入 | 以未来价值而非历史评估 |
