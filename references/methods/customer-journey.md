# 客户旅程地图（Customer Journey Map）

客户旅程地图是将用户与产品/服务的全部接触点可视化的工具，帮助识别体验断点、优化机会和情绪低谷。

---

## 核心框架

一张完整的客户旅程地图包含五个水平层：

```
阶段层   │ 认知  │  考虑  │  购买  │  使用  │  推荐  │
─────────┼───────┼────────┼────────┼────────┼────────┤
行动层   │ 用户在每个阶段的具体行为                    │
接触点   │ 用户接触品牌/产品的渠道和时刻               │
想法层   │ 用户在每个时刻的思考/疑问                   │
情绪层   │ ↗ ↘ ↗ ↗ ↘  情绪曲线（高兴→沮丧→高兴）     │
机会层   │ 发现的改善机会和建议行动                    │
```

---

## 五个旅程阶段详解

### 阶段 1：认知（Awareness）

用户从不知道我们的产品/服务，到第一次听说。

**关键问题**：
- 用户是通过什么渠道第一次知道我们的？
- 在那个时刻，他们处于什么情境？
- 第一印象是什么？

**常见接触点**：
搜索引擎、社交媒体广告、朋友推荐、行业活动、媒体报道

---

### 阶段 2：考虑（Consideration）

用户开始评估，将我们与其他选择进行比较。

**关键问题**：
- 用户会去哪里做研究？
- 他们最在意哪些评估维度？
- 哪些信息能让他们更有信心？
- 什么阻止了他们继续向前？

**常见接触点**：
官网、评测文章、同行推荐、演示视频、竞品对比文章

---

### 阶段 3：购买（Purchase/Decision）

用户做出购买/使用决定。

**关键问题**：
- 最终触发决策的是什么？
- 决策时遇到了什么摩擦？
- 谁参与了决策（尤其 B2B）？
- 付款/签约流程是否顺畅？

**常见接触点**：
定价页、销售对话、试用体验、合同流程

---

### 阶段 4：使用（Onboarding & Retention）

用户开始使用产品，度过学习曲线，形成习惯。

**关键问题**：
- 第一次使用体验是什么感受？
- 用户在哪个环节容易卡住/放弃？
- 什么让他们「啊哈时刻」（Aha Moment）到来？
- 什么让他们流失？

**常见接触点**：
新手引导、帮助文档、客服、邮件通知

---

### 阶段 5：推荐（Advocacy）

满意的用户成为推荐者。

**关键问题**：
- 什么让用户愿意主动推荐？
- 推荐的常见场景和方式？
- 有没有推荐激励机制？

**常见接触点**：
NPS 调研、口碑传播、案例分享、转介绍项目

---

## 客户旅程地图绘制流程

### 步骤 1：选定用户画像（Persona）

旅程地图必须针对特定用户类型，不能一张图代表所有用户。

```
为哪类用户绘制？：[具体 Persona 名称 + 描述]
核心 Job：[这类用户使用产品要完成的核心任务]
```

### 步骤 2：定义旅程范围

```
起点：[用户第一次意识到需求的时刻]
终点：[成为长期用户/推荐者]
关注层：[选择要分析的层（行动/情绪/机会）]
```

### 步骤 3：数据收集

**定性数据来源**：
- 用户访谈（重点：讲述购买/使用故事）
- 客服对话记录
- 销售团队洞察

**定量数据来源**：
- 漏斗数据（各阶段转化率）
- 用户行为数据（热图、点击路径）
- NPS / CSAT 调研
- 流失问卷

### 步骤 4：绘制情绪曲线

情绪曲线是旅程地图最有价值的部分，显示用户在整个旅程中的情绪起伏。

```
情绪曲线（-5 到 +5）：

+5 │                     ★ 啊哈时刻
+4 │         ✓ 找到产品
+3 │ ★ 听说
+2 │
+1 │
 0 │
-1 │           ✗ 看不懂定价
-2 │
-3 │                  ✗ 新手引导太复杂
-4 │
-5 │
   └─────────────────────────────────→ 旅程阶段
      认知    考虑    购买    使用    推荐
```

情绪低谷 = 最大的改善机会

## B2B 旅程地图特殊考量

B2B 购买通常涉及多个决策者，需要绘制**多角色旅程**：

### 关键角色识别

| 角色 | 在旅程中的作用 | 关注点 |
|------|--------------|--------|
| **发起者** | 识别需求，推动评估 | 解决业务问题 |
| **评估者** | 技术/业务评估 | 功能、集成、安全 |
| **决策者** | 最终拍板 | ROI、风险、战略契合 |
| **使用者** | 实际使用产品 | 易用性、效率 |
| **把关者** | 合规/IT审批 | 数据安全、合同 |

**关键原则**：
- 每个角色都有独立的旅程和情绪曲线
- 关注角色之间的信息传递（断点往往在这里）
- 决策者通常在流程后期进入，需要有针对性的沟通材料

---

## 旅程地图 vs. 用户故事地图

| 维度 | 客户旅程地图 | 用户故事地图 |
|------|------------|------------|
| 视角 | 用户体验全链路 | 产品功能规划 |
| 用途 | 发现体验断点、识别机会 | 规划迭代优先级 |
| 输出 | 情绪曲线 + 改善机会 | 功能列表 + Sprint 计划 |
| 最常使用者 | 市场/研究/产品团队 | 产品/工程团队 |
